# Metrics and Learning Loop

## Datenquellen

Im MVP ist Search Console die primäre spätere Datenquelle. Weitere Engagement-Signale dürfen nur genutzt werden, wenn sie sauber erhoben, dokumentiert und mit den Datenschutz- und Trust-Anforderungen vereinbar sind.

## Metriken

- Impressionen.
- Klicks.
- CTR.
- Ranking-Veränderungen.
- Spätere Engagement-Signale, sofern verfügbar und sauber dokumentiert.

## Keine erfundenen Zielwerte

Diese Baseline definiert keine Zielwerte für Traffic, Rankings, CTR, Umsatz oder Conversion. Werte dürfen erst aufgenommen werden, wenn sie aus realen Daten oder expliziten Operator-Vorgaben stammen.

## Review-Entscheidungen

| Entscheidung | Bedeutung |
| --- | --- |
| keep | Artikel erfüllt Zweck und bleibt unverändert |
| improve | Artikel braucht gezielte Verbesserung |
| rewrite | Artikel verfehlt Zweck oder Suchintention deutlich |
| retire | Artikel soll entfernt oder nicht weitergeführt werden |

## Learning Loop

1. Daten nach Veröffentlichung erfassen.
2. Auffällige Artikel identifizieren.
3. Quellen, SERP-Intent und Senior UX erneut prüfen.
4. Entscheidung `keep`, `improve`, `rewrite` oder `retire` dokumentieren.
5. Änderungen wieder durch Review Gate führen.
