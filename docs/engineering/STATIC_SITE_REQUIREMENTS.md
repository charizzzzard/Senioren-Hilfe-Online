# Static Site Requirements

## Ziel

Die spätere Website soll static-first, schnell, wartbar und seniorengerecht sein. Diese Baseline legt keine finale Framework-Entscheidung fest.

## Anforderungen

- Schnelle Ladezeiten.
- Mobile-first Layout.
- Lesbare Typografie.
- Gute Kontraste.
- Klare Navigation.
- Artikelseiten mit Standardstruktur.
- Themen-Hubs für MVP-Cluster.
- Druckansicht für Checklisten und geeignete Artikel.
- Keine Login-Abhängigkeit im MVP.
- Keine Tracking- oder Newsletter-Abhängigkeit im MVP.
- Keine Affiliate- oder Ads-Abhängigkeit im MVP.

## Mögliche spätere Optionen

Spätere technische Optionen können statische Ansätze wie Astro oder Next.js static export prüfen. Diese Baseline entscheidet das nicht final.

## Artikelanforderungen

Artikelseiten müssen später mindestens unterstützen:

- Frontmatter-Metadaten.
- Quellenabschnitt.
- Letzte Prüfung.
- Warnhinweise.
- Angehörigen-Hinweis.
- Druckbare Checkliste, wenn erforderlich.

## Themen-Hubs

Themen-Hubs sollen später Cluster erklären und zu konkreten Problemlösungen führen. Sie sind kein Ersatz für konkrete Schritt-für-Schritt-Artikel.
