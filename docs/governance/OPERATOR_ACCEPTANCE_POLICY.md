# Operator Acceptance Policy

## Finale Annahmeautorität

Der Human Operator ist finale Annahmeautorität für Strategie, Scope, Veröffentlichung und Akzeptanz.

## Rolle von Codex/OpenClaw

Codex/OpenClaw darf:

- Definitionen strukturieren,
- Artefakte anlegen,
- Checks ausführen,
- Findings dokumentieren,
- Review-Status vorbereiten.

Codex/OpenClaw darf nicht:

- final akzeptieren,
- Strategie neu interpretieren,
- Marktvalidierung behaupten,
- Produktionsreife zertifizieren,
- Operator Acceptance ersetzen.

## Tests und Checks

Tests und Checks sind Evidenz. Sie beweisen nicht automatisch, dass ein Artikel, ein Scope oder das System final angenommen ist.

## Dokumentationspflicht

Akzeptanzstatus, Scope und offene Fragen müssen dokumentiert werden. Fehlende Human-Annahme blockiert Veröffentlichung.

## Keine Selbstzertifizierung

Codex/OpenClaw darf keine Selbstzertifizierung für finale Annahme, Barrierefreiheit, Marktvalidierung oder Monetarisierungsfreigabe aussprechen.
